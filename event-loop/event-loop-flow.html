Node.js Event Loop 흐름

Timer, I/O, Microtask, DB 작업이 각자 다른 대기 지점을 거쳐 Event Loop로 돌아오는 흐름을 단계별로 확인한다.

JavaScript Thread

대기

현재 JS 코드를 실행하는 단일 스레드.

libuv

대기

비동기 I/O를 OS나 Thread Pool로 넘긴다.

Network / External API

대기

외부 API 요청은 네트워크와 상대 서버에서 처리된다.

DB Connection Pool

대기

DB 드라이버가 connection 한도 안에서 query를 실행한다.

Microtask Queue

비어 있음

Promise then, await 이후 continuation이 대기한다.

Timer Queue

비어 있음

setTimeout, setInterval callback이 대기한다.

I/O Queue

비어 있음

네트워크, DB 완료 이벤트가 처리 대기한다.

Event Loop

감시 중

Call Stack이 비면 우선순위와 phase에 따라 queue를 처리한다.
